## 📖 项目简介

本项目为亚马逊卖家提供**Chrome浏览器扩展**，实现从CSV/Excel表格批量导入商品数据，一键自动填写亚马逊商品上传表单。

### 🌟 核心优势
- **全自动流程**：自动搜索ASIN -> 自动进入表单 -> 自动识别并填写 -> 自动保存草稿 -> 自动开始下一个。
- **智能导航**：保存成功后直接在当前页面跳转，保存失败自动开新标签页防止卡死。
- **可视化反馈**：已填写字段显示蓝色边框和⚡闪电图标，进度一目了然。
- **零干扰**：只需一次登录，插件在后台静默运行。

## 📥 安装指南

### 方式一：加载已解压的扩展程序 (推荐开发版)
1. 下载项目源码或 `Amazon-auto-listing-v7.1-fixed.zip` 并解压。
2. 打开 Chrome 浏览器，访问 `chrome://extensions/`。
3. 开启右上角 **"开发者模式"**。
4. 点击 **"加载已解压的扩展程序"**。
5. 选择解压后的 `chrome_extension` 文件夹。

### 方式二：CRX 安装 (如已提供)
1. 直接拖拽 CRX 文件到扩展程序页面即可。

## 🚀 使用教程 (保姆级)

### 第一步：准备数据
1. 打开 `wanzhengbiaodan.csv` 模板文件。
2. **切勿修改表头**：第一行英文表头和第二行中文注释请保持原样。
3. **填写商品信息**：从第三行开始填写。
   - **必填项**：ASIN, SKU, 价格, 库存, 品牌, 图片URL等。
   - **图片**：填写网络图片链接 (https://...)。
4. **保存文件**：务必保存为 **CSV (UTF-8)** 格式。

### 第二步：启动插件
1. 登录 [亚马逊卖家后台](https://sellercentral.amazon.com/)。
2. 进入 **"添加新商品"** 页面 (Add a Product)。
   - 美站：`.../product-search`
   - 日站：`.../product-search/search`
3. 此时页面会出现 **悬浮面板**。

### 第三步：开始自动化
1. 在悬浮面板点击 **"导入CSV"**，选择你准备好的文件。
2. 面板会显示 "已加载 X 个商品"。
3. 点击 **"开始全自动上传"**。
4. **喝杯咖啡 ☕**：插件会自动搜索 ASIN，通过 Tab 键切换页面填写所有信息，最后保存草稿并跳转下一个。

## ⚠️ 重要注意事项

### 1. CSV 文件格式
- **严格 88 列**：请勿随意增加或删除列。如果编辑器提示 "Row X has 89 columns"，请检查是否多输入了逗号。
- **编码**：必须是 **UTF-8** 编码，否则中文可能会乱码。

### 2. 浏览器环境
- **保持前台**：建议让浏览器窗口保持在前台，或者不要最小化（可以被其他窗口遮挡）。Chrome 对最小化窗口的资源限制可能导致脚本变慢。
- **不要关闭标签页**：插件运行期间，请勿手动关闭正在填写的标签页。

### 3. 图片上传
- **只支持 URL**：目前仅支持网络图片链接。
- **本地图片**：如果你只有本地图片，请先上传到图床（如 Imgur, sm.ms）获取 URL，或者参考 `IMAGE_UPLOAD_GUIDE.md` 搭建本地服务器。

### 4. 异常处理
- **暂停/继续**：遇到突发情况，可随时点击悬浮面板的 "暂停" 按钮，处理完后点击 "继续"。
- **红色报错**：如果插件提示元素未找到，通常是因为网络卡顿或亚马逊页面改版。插件会自动重试，如果一直失败，请手动辅助一下，然后插件会继续接管。

## 🔧 常见问题 (Q&A)

**Q: 为什么变体页没有填？**
A: v7.1 已经修复了变体页识别问题 (支持 Variation, 多属性等关键词)。如果仍未识别，请检查页面是否加载完全。

**Q: 上传图片失败？**
A: 请检查 URL 是否可以在浏览器直接打开。部分链接可能有防盗链设置，导致插件无法读取图片数据。

**Q: 只有部分字段填了？**
A: 插件会通过蓝色边框标记已填字段。未填的字段通常是非必填项，或者是 CSV 中对应列为空。
