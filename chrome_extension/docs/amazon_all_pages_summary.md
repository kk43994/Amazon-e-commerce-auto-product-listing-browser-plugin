## 📝 开发建议

### 1. 元素定位策略
- **优先使用UID**：每个元素都有唯一的UID标识
- **备用选择器**：
  - textbox类型：使用placeholder属性
  - radio/checkbox：使用文本内容匹配
  - generic selector：点击后等待dropdown出现

### 2. 页面流程控制
```javascript
// 页面切换示例
const pageTabs = {
  productDetails: "UID_5",
  images: "UID_7",
  offer: "UID_9",
  safety: "UID_11"
};

// 必填字段验证
const requiredFields = {
  productDetails: [46_38, 46_42, 46_71, 46_75, 46_79],
  offer: [48_35, 48_53, 48_88],
  safety: [53_30, 53_46, 53_56]
};
```

### 3. 数据映射建议
```javascript
// Excel字段到页面元素的映射
const fieldMapping = {
  "产品标题": "46_38",
  "品牌": "46_42",
  "型号": "46_71",
  "价格": "48_53",
  "库存": "48_35",
  "原产地": "53_30"

```

### 4. 特殊处理注意事项

#### 图片上传
- 支持批量选择和拖放
- 第1张必须设为主图
- 最多9张图片总数限制
- 需要处理上传进度显示

#### 下拉框选择
- 先点击触发下拉框显示
- 等待选项列表加载
- 通过文本匹配选择选项
- 某些下拉框支持搜索过滤

#### 日期选择器
- 点击日历按钮
- 等待日期选择器弹出
- 选择日期或直接输入

#### 动态添加字段
- 多个字段有"添加更多"链接
- 点击后会动态增加输入框
- 需要处理动态元素定位

### 5. 保存策略
- **保存为草稿**：每个页面都可以独立保存
- **最终提交**：需要所有必填字段完成
- **自动保存**：建议实现定时自动保存功能

## ⚠️ 重要提醒
1. **页面必填字段**：带 * 标记的为Amazon系统必填
2. **重要字段**：带 ⭐ 标记的为业务重要字段
3. **提交后不可修改**：某些字段（如外部产品ID、浏览节点）提交后无法更改
4. **FBA优先**：报价页的配送方式默认推荐FBA
5. **语言切换**：所有页面都支持多语言切换
